# DocSage

DocSage is a Streamlit document Q&A app for creating chat sessions, adding documents or web links as sources, and asking questions answered from that source content.

## Features

- Create, open, and delete chat sessions.
- Upload source documents for a specific chat.
- Add web links and index their page text.
- Store chats, messages, and source metadata in SQLite.
- **PDFs**: local structured Q&A via OCR (for scanned pages), PyMuPDF4LLM section extraction, and llama.cpp (Mozilla structured-qa blueprint).
- **Other sources**: per-chat Chroma embeddings with Gemini RAG for links and non-PDF files.
- Stream assistant responses with a simple typing effect.

## Prerequisites

- Python 3.10 or newer.
- A Google Gemini API key (for links and non-PDF document RAG).
- A local GGUF model file for llama.cpp PDF Q&A (recommended: Qwen2.5-3B-Instruct).
- ~10 GB RAM recommended for local PDF Q&A.
- Tesseract OCR (required for scanned/image PDFs):
  ```bash
  # macOS
  brew install tesseract

  # Ubuntu/Debian
  sudo apt install tesseract-ocr
  ```
- `pip` for installing Python packages.

## Installation

From the project root, install dependencies:

```bash
pip install -r requirements.txt
```

On macOS, if `llama-cpp-python` fails to install, try:

```bash
CMAKE_ARGS="-DGGML_METAL=on" pip install llama-cpp-python
```

If you do not have `requirements.txt`, install the dependencies manually:

```bash
pip install streamlit langchain langchain-google-genai langchain-chroma langchain-community langchain-text-splitters beautifulsoup4 requests django-environ docx2txt unstructured structured-qa ocrmypdf
```

## Configuration

Create a `.env` file in the project root:

```bash
GOOGLE_API_KEY=your_gemini_api_key_here
LLAMA_MODEL_PATH=path/to/model.gguf
```

`LLAMA_MODEL_PATH` can be a local GGUF file path or a HuggingFace URI supported by structured-qa.

Example model (Mozilla recommendation):

```bash
# Download a Qwen2.5-3B-Instruct GGUF, then set:
LLAMA_MODEL_PATH=/path/to/Qwen2.5-3B-Instruct-f16.gguf
```

The app reads environment variables with `django-environ`.

Do not commit `.env` or real API keys.

## Database Setup

Initialize the SQLite database once before running the app:

```bash
python create_relational_db.py
```

This creates `doc_sage.sqlite` with tables for chats, sources, and messages.

## Usage

Start the Streamlit app:

```bash
streamlit run chats.py
```

Then use the browser interface to:

1. Create a chat.
2. Upload a PDF (local structured sections) or other document / web link (Chroma + Gemini).
3. Ask questions related to the uploaded or linked source content.

## Supported Sources

Document uploads support:

- `.pdf` — OCR for image-only pages (via `ocrmypdf`), then structured section workflow (local llama.cpp, no Chroma)
- `.txt`, `.docx`, `.csv`, `.html`, `.md` — Chroma RAG with Gemini

Web links are fetched with `requests`, parsed with BeautifulSoup, and indexed in Chroma with Gemini embeddings.

## Project Structure

| Path | Purpose |
| --- | --- |
| `chats.py` | Streamlit entrypoint, UI, routing, uploads, links, and chat flow |
| `db.py` | SQLite CRUD helpers |
| `vector_functions.py` | Non-PDF document loading, Chroma vector store, Gemini RAG |
| `structured_functions.py` | PDF preprocessing and llama.cpp find/retrieve/answer workflow |
| `create_relational_db.py` | One-time SQLite schema setup |
| `requirements.txt` | Python dependencies |
| `doc_sage.sqlite` | Local SQLite database generated by setup/runtime |
| `sections/` | PDF section files per chat (`sections/chat_{chat_id}/{doc_name}/`) |
| `persist/` | Local Chroma vector store (non-PDF sources only) |
| `temp_files/` | Temporary upload directory |

## Q&A Pipelines

### PDF sources (local-first)

- OCR: `ocrmypdf` with `skip_text` — text PDFs are unchanged; image-only pages are OCR'd locally via Tesseract
- Preprocessing: `structured-qa` / PyMuPDF4LLM converts PDFs to markdown sections
- Storage: `./sections/chat_{chat_id}/{doc_name}/`
- Q&A: Mozilla find/retrieve/answer loop via llama.cpp
- No API key required for PDF answers

### Links and non-PDF files (Gemini RAG)

- Chroma persistence directory: `./persist`
- Collection naming pattern: `chat_{chat_id}`
- Text splitter: `CharacterTextSplitter(chunk_size=1000, chunk_overlap=0)`
- Retriever: `similarity_score_threshold`
- Default score threshold: `0.6`
- Chat model: `gemini-2.5-flash`
- Embeddings: `gemini-embedding-001`

### Mixed-source chats

If a chat has both PDFs and other sources, answers are returned in two labeled sections: one from PDF sources (llama.cpp) and one from other sources (Gemini RAG).

## Migration note

If you previously indexed sources with OpenAI embeddings, delete `./persist/` and re-upload non-PDF sources after switching to Gemini.

## Development Notes

- This is a small Streamlit app, not a packaged Python library.
- Run the test suite with pytest (no API keys or local model required for unit tests):

```bash
pip install -r requirements-dev.txt
pytest
```

- Pre-commit runs ruff, mypy, and pytest when dev dependencies are installed (`pre-commit run --all-files`).
- Use a quick syntax check after Python edits:

```bash
python -m py_compile chats.py db.py vector_functions.py structured_functions.py create_relational_db.py
```

- Avoid committing runtime files such as `.env`, `doc_sage.sqlite`, `persist/`, `sections/`, `temp_files/`, and `__pycache__/`.
