# DocSage

DocSage is a Streamlit RAG chatbot for creating chat sessions, adding documents or web links as sources, and asking questions answered from that source content.

## Features

- Create, open, and delete chat sessions.
- Upload source documents for a specific chat.
- Add web links and index their page text.
- Store chats, messages, and source metadata in SQLite.
- Store per-chat embeddings in Chroma collections.
- Generate answers with LangChain and OpenAI `gpt-4o-mini`.
- Stream assistant responses with a simple typing effect.

## Prerequisites

- Python 3.10 or newer.
- An OpenAI API key.
- `pip` for installing Python packages.

## Installation

From the project root, install dependencies:

```bash
pip install -r requirements.txt
```

If you do not have `requirements.txt`, install the dependencies manually:

```bash
pip install streamlit langchain langchain-openai langchain-chroma langchain-community langchain-text-splitters beautifulsoup4 requests django-environ pypdf docx2txt unstructured
```

## Configuration

Create a `.env` file in the project root:

```bash
OPENAI_API_KEY=your_openai_api_key_here
```

The app reads environment variables with `django-environ`.

Do not commit `.env` or real API keys.

## Database Setup

Initialize the SQLite database once before running the app:

```bash
python create_relational_db.py
```

This creates `doc_sage.sqlite` with tables for chats, sources, and messages.

## Usage

Start the Streamlit app:

```bash
streamlit run chats.py
```

Then use the browser interface to:

1. Create a chat.
2. Upload a document or add a web link.
3. Ask questions related to the uploaded or linked source content.

## Supported Sources

Document uploads support:

- `.txt`
- `.pdf`
- `.docx`
- `.csv`
- `.html`
- `.md`

Web links are fetched with `requests`, parsed with BeautifulSoup, and indexed as source text for the active chat.

## Project Structure

| Path | Purpose |
| --- | --- |
| `chats.py` | Streamlit entrypoint, UI, routing, uploads, links, and chat flow |
| `db.py` | SQLite CRUD helpers |
| `vector_functions.py` | Document loading, Chroma vector store, retriever, and RAG answer generation |
| `create_relational_db.py` | One-time SQLite schema setup |
| `requirements.txt` | Python dependencies |
| `doc_sage.sqlite` | Local SQLite database generated by setup/runtime |
| `persist/` | Local Chroma vector store |
| `temp_files/` | Temporary upload directory |

## RAG Details

- Chroma persistence directory: `./persist`
- Collection naming pattern: `chat_{chat_id}`
- Text splitter: `CharacterTextSplitter(chunk_size=1000, chunk_overlap=0)`
- Retriever: `similarity_score_threshold`
- Default score threshold: `0.6`
- Chat model: `gpt-4o-mini`

## Development Notes

- This is a small single-module Streamlit app, not a packaged Python library.
- There is no test suite, linter, formatter, or typechecker configured.
- Use a quick syntax check after Python edits:

```bash
python -m py_compile chats.py db.py vector_functions.py create_relational_db.py
```

- Avoid committing runtime files such as `.env`, `doc_sage.sqlite`, `persist/`, `temp_files/`, and `__pycache__/`.
